This is a list of the many disparate projects I have worked on. I've tried my best to sort it by kind and age, but some things reject all attempts at classification.

HPCMy current main project is sunwalker, a new-gen testing system for programming contests:

  • sunwalker-box (2022—2023) — An isolation box, more performant than anything anyone has ever done
  • sunwalker-image (2022) — A tiny Docker-like image containing lots of runtimes, compilers, and interpreters

My former activity in the competitive programming sphere has influenced my goals and understanding of good software, which is why I love pushing the limits:

  • btccaserestore (2024) — A performant brute-force utility to restore Bitcoin addresses from lowercased strings
  • Feijoa (2024) — An efficient CRC-based hash-like cryptographic primitive (that's not an oxymoron!)
  • blazingio (2024) — 5x faster drop-in <iostream> replacement for competitive programming
  • taco (2022) — An experiment to see how well we can automatically perform domain-specific compression (decompression isn't supported though, so...)
  • crossmist (2023) — Efficient (read as: 100s of unsafe) and seamless cross-process communication for Rust

This dedication has brought me to demoscene and retrocomputing. I developed stuff for Electronica BK once:

  • Good Apple (2018) — A demo (see YouTube), squeezing the most out of the low-level features, manual RAM and disc control
  • pdpy11 (2018—2022) — A modern PC-based macro assembler for the PDP-11 architecture with IDE integration and pretty errors
  • lzsa3 (2021) — A modification of LZSA2 for faster decoding on PDP-11 with a decompressor provided

I also host a programming-related site:

  • reg.algocode.ru (2021—2023) — Standings for various local programming competitions, automatically pulling results from many distinct sources (this has become the go-to site for many school students in Russia)

Cursed stuffI've made quite a few things just for fun:

  • Steam (2024) — 3 KB React-like framework with JSX-like syntax, not requiring any custom build system (similar to Preact and htm)
  • docker-boot (2024) — Replace your running Linux system with a Docker container
  • disas (2023) — A beautiful and slow binutils-based disassembler in Bash
  • win-sudo (2017—2018) — sudo for Windows
  • BatchBuilder (2018) — OOP in Batch (yes, I am mentally ill)

CybersecurityI sometimes take part in CTF competitions. I'm also one of the developers of UgraCTF and developed some closed-source software for it:

  • kyzylborda-box (2022) — A lightweight Docker/Podman alternative for low-latency parallel deployment of 100s of containers
  • kyzylborda-lib (2022—2023) — A framework for CTF tasks

DeploymentI hosted various competitive programming competitions in Moscow and got my hands dirty with mass deployment of a combined Windows + Linux image in... unpleasant conditions.

  • broadband (2022) — Simple reliable broadcast over UDP (I had to develop it in a day to make the deadline)

Online servicesI develop habra.chat, a traditional New Year chat for Habr. It has grown from a single SSH chat to a mirrored experience synchronized among many networks.

  • ssh-chat — An extended fork of the backend that makes the chat accessible via SSH
  • NickServ (2021—2022) — Nick registration
  • webchat (New Year Eve of 2023) — Web frontend, see live

ZeroNetI used to be active on the decentralized network ZeroNet as gitcenter for a large portion of my life and, among other things, developed a few services for the network, the most popular ones being:

  • Git Center (2017—2018) — A decentralized GitHub alternative supporting Git and Mercurial repositories
  • Kiwipedia (2018) — A wiki for ZeroNet with support of mirroring Wikipedia
  • Name.YO (2019—2021) — An alternative DNS provider-like service

I was one of the lead developers of the network and developed a few plugins for it, bringing real-time communication to ZeroNet:

  • 0Play (2018) — The first demonstration of the capabilities: a chat, chess and tic-tac-toe games
  • Instant Messenger (2018) — An unfinished Telegram/WhatsApp-like alternative
  • PeerMessage (2018—2019) — A plugin enabling decentralized real-time communication
  • StreamZ (2018—2019) — A Twitch-like video streaming service
  • zIRC (2019) — A simple closed-group IRC-like chat
  • ZeroFile (2017) — Simple file storage
  • CodeR (2017—2018) — JSFiddle/CodePen alternative with client-side CSS/JS transpiling support
  • ZeroMailProxy (New Year of 2018) — A tool for sending and receiving ZeroNet mail via a local SMTP/POP server
  • Zero Management System (2018) — An unfinished CMS
  • /z/place (2018) — An ugly clone of r/place
  • Gopher (2018) — An alternative gopher frontend to zites supporting the protocol
  • ThunderProxy (2018) — A tool for accessing ThunderWave with a local IRC server
  • BackgroundProcessing (2018—2020) — An attempt to enable efficient server-side computations by running user-provided Python code in a sandbox
  • Ark (2019) — A plugin bringing backwards-compatible multi-cryptography capabilities to the official ZeroNet client
  • ZeroTalk-Matrix (2019) — A bridge between Matrix and ZeroTalk

I also wrote a few libraries some people found useful:

  • ZeroDevLib (2017—2019) — User-friendly JavaScript libraries for ZeroNet APIs
  • ZeroNet-cmd-lib (2018) — Command-line utilities
